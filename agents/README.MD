# 🤖 OpenCode Agents Ecosystem

Цей документ описує кастомних AI-агентів, налаштованих для локального середовища розробки (Ubuntu 22.04) та роботи над проєктом. 

Усі агенти налаштовані відповідати українською мовою, генеруючи код та технічну документацію англійською. Конфігурації агентів знаходяться у `~/.config/opencode/agents/` (або `.opencode/agents/` у корені проєкту).

---

## 🌟 Основний агент (Primary)

Цей агент працює за замовчуванням при відкритті чату.

* **`build`** (Глобальний системний промпт)
    * **Роль:** Elite AI Software Engineer.
    * **Спеціалізація:** Написання коду, дебагінг, загальна розробка.
    * **Контекст:** Розуміє повний стек (Laravel, Vue 3, Inertia, Go, PostgreSQL, Redis, RabbitMQ) та інфраструктуру (Docker/Vagrant). 

---

## 🎯 Субагенти (Subagents)

Викликаються через `@назва-агента` у чаті OpenCode. Усі субагенти працюють у режимі захисту від випадкових змін (дозволи на редагування та виконання критичних bash-команд встановлені на `ask`).

### 🏗 Архітектура та Планування
* **`@architect`**
    * **Роль:** Staff-level Software Architect.
    * **Задачі:** Проєктування бази даних (PostgreSQL), взаємодії мікросервісів (RabbitMQ/Redis), вибір патернів DDD, розробка API та оптимізація структури Inertia.js.
* **`@task-planner`**
    * **Роль:** Technical Product Manager / Scrum Master.
    * **Задачі:** Декомпозиція великих фіч на Jira-тікети. Створення User Stories (у форматі BDD), визначення Acceptance Criteria та оцінка складності у Story Points.

### 🕵️ Code Review
* **`@laravel-reviewer`**
    * **Роль:** Senior Laravel Reviewer.
    * **Задачі:** Аудит PHP-коду. Перевірка на N+1 проблеми, безпеку, SOLID, правильне використання ресурсів бази даних та архітектурних патернів Laravel.
* **`@go-reviewer`**
    * **Роль:** Senior Golang Engineer.
    * **Задачі:** Аналіз Go-коду на ідіоматичність. Пошук витоків пам'яті (goroutine leaks), стану гонки (data races) та перевірка правильного менеджменту контекстів (`context.Context`).

### 🧪 Тестування (QA Automation)
* **`@laravel-test-writer`**
    * **Задачі:** Генерація бекенд-тестів за допомогою Pest PHP. Покриття контролерів (включно з Inertia responses), сервісних класів, моделей та перевірка фонових задач (черг).
* **`@inertia-vue-tester`**
    * **Задачі:** Генерація фронтенд-тестів. Пише ізольовані юніт-тести через Vitest/Vue Test Utils для компонентів, а також E2E тести (Laravel Dusk / Cypress) для перевірки реальних користувацьких сценаріїв.

### ⚙️ Інфраструктура та Інструменти
* **`@devops`**
    * **Задачі:** Робота з інфраструктурою. Написання `Dockerfile` та `docker-compose.yml` для продакшену, налаштування Vagrant + Ansible для локальної розробки, побудова CI/CD пайплайнів та конфігурація New Relic / Grafana.
* **`@nvim-expert`**
    * **Задачі:** Допомога з кастомним конфігом Neovim (v0.11.3+). Робота з менеджером `lazy.nvim`, використання сучасного Lua API, дебагінг плагінів та правильне створення keymaps без використання застарілого Vimscript.
